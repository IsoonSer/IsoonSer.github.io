เผยแพร่ Blog ลงบน Website ด้วย Hugo

ลองอะไรใหม่ๆ สำหรับผม คนที่ไม่ถนัดในการทำ UI ให้เว็บ

April 15, 2025

  • hugo

        เนื่องจากว่าผมอยากจะมีพื้นที่ที่เอาไว้สำหรับบอกเล่า แบ่งปันเรื่องราว แบ่งปันประสบการณ์มานานแล้ว แต่ก็เนื่องจากในชีวิตมีอะไรต้องทำเยอะเกิน บวกกับการผัดวันประกันพรุ่งของตัวเอง ก็เลยไม่ได้มีโอกาสได้ทำสักที มาถึงวันนี้ ผมเรียนจบ ป.ตรี แล้วครับ เย่ …ก็เลยเป็นโอกาสอันดีที่จะได้เริ่มต้นลองอะไรใหม่ๆ ในช่วงเวลานี้

        อะ เข้าเรื่องกันเลยดีกว่า ก็เนื่องจากว่าผมค่อนข้างที่จะไม่ enjoy กับการออกแบบ Figma ในการทำ UI ให้ Web รวมถึงไม่ค่อยชอบการเขียน CSS ผมก็เลยพยายามที่จะหาหนทางมีเว็บให้ตัวเอง แบบที่ไม่ต้องยุ่งกับสิ่งพวกนี้ โดยก่อนหน้านี้ ประมาณปีที่แล้ว ผมก็ได้ไปเห็น Web Blog ที่ชื่อว่า suksit dot com แล้วผมก็ได้ลองอ่านงานเขียนของพี่เค้าหลายอันพอสมควร แต่ก็ไม่ได้เอะใจอะไร จนกระทั่งมาวันหนึ่ง ก็เหลือบไปเห็นคำว่า Powered by Hugo บนหน้าแรกของเว็บพี่เค้า

        ผมก็เลยลองกดเข้าไปดู ว่ามันคืออะไร แล้วก็ได้ลองค้นหาข้อมูล ก็เลยไปพบว่า มันเป็นเครื่องมือที่ช่วยให้เราสร้าง Static Web ได้ง่าย โดยการไปนำ Template ของคนอื่นมาดัดแปลง แค่นี้เลย ผมก็เลยสนใจ ทีนี้ก็ได้ลองไปค้นหาข้อมูลใน Youtube เพื่อไปดูว่าชาวบ้านเขาใช้สิ่งนี้ทำอะไรบ้าง ก็บังเอิญไปพบเจอกับคลิปวีดีโอของคุณพี่ NetworkChuck ละผมก็ชอบดูคลิปพี่แกอยู่แล้วด้วย เพราะว่าเป็นช่องที่ทำอะไรๆ เกี่ยวกับ IT หลายๆ อย่าง ได้น่าสนใจดี ก็เลยลองนั่งดูคลิปจนจบ ก็พบว่า จุดประสงค์ของพี่แก ก็คือ แกอยากจะมีพื้นที่เอาไว้ถ่ายทอดเรื่องราวของแกเอง ให้คนอื่น ลูกหลาน ได้รับการถ่ายทอดประสบการณ์จากชีวิตของพี่แก แล้วแกก็สอนการใช้งาน Hugo ว่าจะเอามาทำเป็น Web Blog ของเราได้อย่างไร

        ผมก็เลยได้ลองทำตาม ลองปรับแต่งดู ก็เลยเป็นที่มาของ Blog นี้ โดย Blog นี้ ผมก็จะมาเขียนสรุปวิธีการใช้งาน Hugo สร้าง Web Blog ว่ามีขั้นตอน อะไรอย่างไร (รวมถึงจะมีแนะนำการใช้ bash script ในการช่วยอัพเดต Content จาก Obsidian ให้ไป Sync กับตัว Web Blog ที่จะถูก Publish บน Github Page ด้วย) รวมถึงแนะนำการเอา Web Blog ที่สร้างจาก Hugo นี่แหละ ไป Host บน Github Page เพื่อที่ว่าชาวโลก จะได้เข้ามาอ่านงานของเราได้ …. โอเค หลังจากที่เล่ามายืดยาว เราก็มาดูวิธีการใช้ Hugo ในการสร้าง Web Blog ของผมกันเลย

        อันดับแรกก็เข้าไปดาวน์โหลด hugo จาก website กันก่อน โดยเข้าไปดูวิธีการดาวน์โหลดและติดตั้งได้จาก link นี้ โดยส่วนตัวผมแล้ว ผมใช้ macbook ก็จะเลือกวิธีการติดตั้งโดยใช้ homebrew เพื่อความสะดวก เมื่อติดตั้งแล้ว ก็ลองใช้คำสั่ง hugo version เพื่อตรวจสอบให้แน่ใจว่าคำสั่ง hugo สามารถรันได้บนเครื่องของเราแล้ว

brew install hugo
hugo version

หลังจากนั้นก็ใช้ hugo command บน command line เพื่อทำการสร้าง hugo project ขึ้นมาใหม่ (ผมใช้ชื่อ project ว่า zoneblog)

hugo new site zoneblog

เมื่อรันคำสร้างในการสร้าง project เสร็จเรียบร้อยแล้ว เราก็จะได้ hugo project ขึ้นมาใหม่ โดยจะมีโครงสร้างไฟล์และโฟลเดอร์ประมาณนี้

.
└── zoneblog
    ├── archetypes
    │   └── default.md
    ├── assets
    ├── content
    ├── data
    ├── hugo.toml
    ├── i18n
    ├── layouts
    ├── static
    └── themes

สิ่งถัดไปที่ต้องทำก็คือ เข้าเลือก Theme บนเว็บไซต์ของ hugo ว่าอยากจะได้ Theme แบบไหนมาใช้กับงานของเรา โดยบนเว็บของ hugo ก็จะมี Theme ให้เลือก หลากหลายรูปแบบ ทุกคนสามารถเข้าไปเลือก Theme ที่ต้องการได้จาก link นี้

เมื่อเลือก Theme ที่ต้องการได้แล้ว ถัดมาก็ให้ติดตั้ง Theme ลงบนงานของเรา (ลงใน directory ของ project เรา นั่นแหละ) โดยในการติดตั้ง Theme ลงบนงานของเรา สามารถดูวิธีการได้จาก Documentation ของ Theme ที่เราเลือกได้เลย ผมจะขอยกตัวอย่าง Theme ที่ผมเลือกมาใช้งาน นั่นคือ Theme ที่มีชื่อว่า Gokarna โดยสามารถดูวิธีการติดตั้งและใช้งานได้จาก Github Project ตัวนี้

git init
git submodule add https://github.com/gokarna-theme/gokarna-hugo.git themes/gokarna

แล้วก็ลองเอา Basic Configuration ไปใส่ในไฟล์ hugo.toml ลองดู

baseURL = "https://example.org/"
defaultContentLanguage = "en"
# Automatically generate robots.txt
enableRobotsTXT = true
languageCode = "en"
theme = "gokarna"
title = "My New Hugo Site"

[menu]
  [[menu.main]]
    # Display name
    name = "Posts"

    # Relative URL slug (appended to baseURL)
    url = "/posts/"

    # Lower weights are listed first in the menu (leftmost); higher weights are
    # listed last in the menu (rightmost)
    weight = 1

  [[menu.main]]
    name = "Tags"
    url = "/tags/"
    weight = 2
    
  [[menu.main]]
    # Unique identifiers are required for menu entries without a name property,
    # or for menu entries which re-use a name
    identifier = "github"

    # Absolute URL to external resource
    url = "https://github.com"
    weight = 3
    
    # Surround the menu entry (or name) with HTML content, such as Feather
    # icons: https://feathericons.com
    pre = "<span data-feather='github'></span>"
    post = ""

ถัดมา ในโฟลเดอร์ Project ของเราให้รันคำสั่ง hugo server แล้วลองเข้าไปที่ http:localhost:1313 ก็จะพบกับ Hugo Site แบบง่ายๆ ของเรา

ส่วนวิธีการเพิ่มรูป Profile เพิ่ม Icon Link ต่างๆ สามารถศึกษาเพิ่มเติมได้จากตัวอย่างเว็บไซต์ ที่เค้าจะมีเขียน Configuration ตัวอย่างให้ ซึ่งในกรณีของ Theme ผม สามารถดูได้ที่ Github นี้ เดี๋ยวผมจะพามาดูวิธีการเขียน Configuration File ที่ผมใช้ในการกำหนดค่าให้ Profile ที่ผมใช้นะครับ (ในไฟล์ hugo.toml)

baseURL = "https://isoonser.github.io/"
defaultContentLanguage = "en"
enableEmoji = true
languageCode = "en"
theme = "gokarna"
title = "OzoneBlog"

[params]
  avatarAltText = "avatar"
  avatarSize = "size-m"
  avatarURL = "/images/my-profile.png"
  description = "Passionate about IT, always seeking to learn more about life, and striving for peace."
  footer = "Isoon Sermsri"
  metaKeywords = ["blog", "gokarna", "hugo"]
  showBackToTopButton = true # true or false for "back to top" button on posts and pages

  socialIcons = [
    {name = "instagram", url = "https://example.com"},
    {name = "facebook", url = "https://www.facebook.com/ozone.isoon"},
    {name = "email", url = "mailto:example@example.com"}
  ]

[menu]
  [[menu.main]]
    name = "Home"
    pre = "<span data-feather='home'></span>"
    url = "/"
    weight = 1

  [[menu.main]]
    name = "Posts"
    pre = "<span data-feather='book'></span>"
    url = "/posts/"
    weight = 2

  [[menu.main]]
    name = "Tags"
    pre = "<span data-feather='tag'></span>"
    url = "/tags/"
    weight = 4

สำหรับการเอารูปไปวางเพื่อใช้ใน Project ของเรา ให้เอาไปวางไว้ที่ static/images ในกรณีของผมก็ให้ตั้งค่า path ของรูป ไปชี้ที่รูปของเรา ที่เราไปวางเอาไว้ ดังนี้ avatarURL = "/images/my-profile.png" (ตั้งค่าบนไฟล์ hugo.toml)

ตัว hugo ก็จะอัพเดต Content ให้เราใหม่ ให้เราลองไปดูที่ localhost:1313 อีกรอบ ก็จะพบว่าหน้า Profile ของเรา ได้เปลี่ยนไปแล้ว

แต่เมื่อลองกดบนแถบด้านบนไปที่ Posts เข้าไป ก็จะพบหน้าต่างนี้ เนื่องจากว่าเรายังไม่ทำการเพิ่ม Post ไปบนเว็บไซต์ของเรา

ต่อไป ผมจะมาลองพาทำการเพิ่ม Post ไปที่เว็บไซต์ของเรา โดยในการสร้าง Post บน Hugo นั้น ต้องเขียนโดยใช้ format เป็น markdown โดยให้เอา Content ในนี้ ไปเพิ่มที่ไฟล์ content/posts/test.md (เข้าไปเอา Content ได้จาก Link นี้)

---
title: "Markdown Syntax Guide"
date: 2019-06-19
description: "Sample article showcasing basic Markdown syntax and formatting for HTML elements."
tags: ["markdown", "css", "html"]
type: post
weight: 20
showTableOfContents: true
---

This article offers a sample of basic Markdown syntax that can be used in Hugo content files, also it shows whether basic HTML elements are decorated with CSS in a Hugo theme.

พอลองไปที่ URL http://localhost:1313/posts/ กับ http://localhost:1313/posts/test/ ก็จะพบผลลัพธ์ดังนี้

เท่านี้ก็เป็นการสร้าง Web Blog แบบง่ายๆ ด้วย Hugo เสร็จเรียบร้อยครับ

โดยถัดมาผมจะพามาดูขั้นตอนการเอา Website ที่สร้างจาก Hugo ขึ้นไปไว้บน Github Page เพื่อที่ว่าชาวโลกจะได้เข้ามาดูงานที่เราเผยแพร่ได้
